아래 지침에 따라 Linux 지식 베이스 문서를 새로 작성해 주세요.

## 입력 정보
- 사용자 입력: $ARGUMENTS
- 첫 번째 인자 = 카테고리 (networking, storage, system, security)
- 두 번째 인자 이후 = 주제 (하이픈 구분)

## 파일 생성 규칙
1. 파일명: `linux-{주제}.md` (소문자, 하이픈 구분)
2. 저장 위치: `{카테고리}/` 디렉토리
3. `rules/doc-writing.md`의 문서 작성 규칙 준수
4. `rules/linux-conventions.md`의 코드 작성 규칙 준수
5. `rules/security-checklist.md`의 보안 체크리스트 통과

## 문서 구조 (반드시 아래 섹션 모두 포함)

```markdown
# {주제명}

## 1. 개요
(이 도구/파일/개념이 무엇이고 왜 중요한지 2~4줄)

## 2. 설명
### 2.1 핵심 개념
### 2.2 실무 명령어
### 2.3 클라우드/DevOps 연계

## 3. 자주 하는 실수
(표 형식: 잘못된 방법 → 올바른 방법 → 이유)

## 4. 트러블슈팅

## 5. TIP
```

문서 작성 완료 후 `CLAUDE.md`의 카테고리별 파일 목록에 항목을 추가해 주세요.
