아래 기준으로 `$ARGUMENTS` 문서를 검토하고 개선점을 제안해 주세요.

## 검토 항목

### 1. 구조 완성도
- [ ] 3개 필수 섹션(개요/설명/자주하는실수) 모두 존재하는지
- [ ] 트러블슈팅 섹션이 있는지

### 2. 코드 품질
- [ ] 모든 명령어에 `#` 한국어 주석이 있는지
- [ ] 언어 태그(`bash`, `ini`, `yaml`)가 있는지
- [ ] 위험한 명령어에 `> **주의**` 경고가 있는지

### 3. 실무 적용성
- [ ] AWS/클라우드 환경 연계 내용이 있는지
- [ ] Terraform/Ansible 예제가 포함됐는지 (해당되는 경우)

### 4. 자주 하는 실수
- [ ] 표 형식으로 작성됐는지
- [ ] 최소 3개 이상의 실수 케이스가 있는지

## 출력 형식

```
## 검토 결과: {파일명}

### 잘된 점
- ...

### 개선 필요 사항
1. [섹션명] 문제 → 개선 방안

### 보안 이슈 (있을 경우)
- 위험도: High/Medium/Low
- 내용: ...
```
