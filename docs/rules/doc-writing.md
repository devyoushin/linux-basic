# 문서 작성 규칙 (Doc Writing Rules)

Claude가 이 저장소에서 문서를 작성하거나 수정할 때 반드시 따라야 할 규칙입니다.

---

## 1. 언어 및 문체

- **기본 언어**: 한국어
- **기술 용어**: 명령어, 파일명, 옵션은 영어 그대로 사용
  - 예: `iptables`, `/etc/fstab`, `--verbose`
- **문체**: 간결한 서술체 (`~다.`, `~한다.`)
- **경어체 금지**: 기술 문서 특성상 `~합니다` 사용하지 않음

## 2. 코드 블록 규칙

- **언어 태그 필수**:
  - ` ```bash ` — shell 명령어
  - ` ```ini ` — 설정 파일
  - ` ```yaml ` — YAML 파일
  - ` ```hcl ` — Terraform
  - ` ```python ` — Python 스크립트
- **주석 필수**: 모든 명령어에 `#` 한국어 주석 추가
- **위험 명령어**: `rm -rf`, `dd`, `mkfs` 등에는 `> **주의**` 경고 블록 추가
- **플레이스홀더**: `<DEVICE>`, `<MOUNT_POINT>` 형식

## 3. 섹션 구성 필수 요소

새 문서는 반드시 아래 섹션을 포함해야 함:

| 섹션 | 내용 | 생략 가능 여부 |
|------|------|----------------|
| 1. 개요 | 무엇인지, 왜 필요한지 | 불가 |
| 2. 설명 | 개념 + 명령어 + IaC 예제 | 불가 |
| 3. 자주 하는 실수 | 표 형식 실수 → 올바른 방법 | 불가 |
| 4. 트러블슈팅 | 증상 → 원인 → 해결 | 가능 |
| 5. TIP | 현장 팁 | 가능 |

## 4. 자주 하는 실수 섹션 형식

반드시 표(table) 형식으로 작성:

| 잘못된 방법 | 올바른 방법 | 이유 |
|-----------|-----------|------|
| `명령어1` | `명령어2` | 이유 설명 |

## 5. 금지 사항

- 추측성 내용 작성 금지 (실제 경험/man 페이지 기반만)
- 주석 없는 코드 블록 금지
- 실제 서버 IP/도메인/계정 정보 노출 금지
