# 보안 검토 체크리스트

Linux 문서 작성 시 반드시 확인해야 할 보안 항목입니다.

---

## 1. 명령어 보안

- [ ] `rm -rf /` 또는 유사한 위험 명령어에 주의 경고 추가
- [ ] 실제 서버 IP/도메인/자격증명 노출 금지
- [ ] root 계정 직접 사용 지양, sudo 활용 권장

## 2. 권한 설정

- [ ] 파일/디렉토리 권한 `777` 사용 시 주의 경고
- [ ] SUID/SGID 비트 설정 시 위험성 명시
- [ ] `/etc/sudoers` 직접 수정 지양, `visudo` 사용 권장

## 3. 네트워크 보안

- [ ] 모든 포트 오픈(`0.0.0.0:*`) 설정에 경고
- [ ] iptables ACCEPT all 정책에 주의 문구
- [ ] 평문 전송 프로토콜(FTP, Telnet) 사용 금지 권장

## 4. 스크립트 보안

- [ ] `curl | bash` 패턴 사용 금지 (스크립트 검증 후 실행 권장)
- [ ] 환경변수에 패스워드 직접 노출 금지
- [ ] `set -euo pipefail` 없는 스크립트에 취약성 언급

## 5. 금지 표현

- 실제 패스워드, API 키, 토큰 예시 작성 금지
- 실제 서버 이름, IP, 도메인 노출 금지
- 모든 민감 정보는 `<YOUR_VALUE>` 형식으로 대체
